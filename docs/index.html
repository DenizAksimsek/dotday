A human readable calendar format.

Compiles to iCalendar (.ics) files.

One file = one day.

Designed for ease of parsing, both by humans and by computers.

The GitHub repo contains an informal description and a reference implementation.

Example 

; 2020-07-13.day -- the filename is the date
; comment
; The title is optional.
13 July (Monday)
================

; Events are separated by blank lines
; !1h -> Your calendar app will remind you 1 hour before the event
10:00 -- 11.50 !1h
MATH 102
@ DB18
i hate this course
so much
~ Mike Oxmall <mike.oxmall@example.edu>

FAQ (Fairly-likely-to-be-Asked Questions)

"Why did you make this?"
— Someone, eventually

Because I couldn't believe nobody else had made it. I wanted a text based calendar that I could put on my desktop and phone, but I also wanted notifications on my phone (text files are too polite to bother you like that). In the end, .day is todo.txt but for calendaring.
